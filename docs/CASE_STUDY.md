# 護家神山 Web App — Case Study

> UI/UX Designer · 互動原型 · Material Design 3 + HeroUI v3 + Tailwind CSS v4

---

## 1. Metadata（基礎專案元數據）

* **Project:** 護家神山 Web App
* **Role:** UI/UX Designer
* **Timeline:** 一日（目標需求定義 → 資訊架構 → Markdown 文件 → UI 設計 → 部署）
* **Tech Stack / UI System:** Material Design 3 + HeroUI v3 + Tailwind CSS v4
* **Platform:** 響應式 Web App（Mobile-first）；Compact 底部導覽／Medium+ 頂部 Pill 導覽
* **Primary Navigation:** 總覽 · 理賠 · AI 顧問 · 保障（4 Tab）；個人資料由右上角頭像進入

---

## 2. Problem & Target（問題與目標）

* **Pain Point:** 當沒有透過家庭的視角檢視保障缺口時，容易疏忽潛在破口，導致家庭經濟出現狀況。
* **Design Goal:** 幫助台灣家庭（特別是多代同堂、高齡化家庭）輕鬆、視覺化地規劃與管理全家人的保障缺口。

---

## 3. Core UX Logic & Laws（核心 UX 邏輯與法則）

### Nielsen 可用性原則：讓家人「看畫面就懂」，不用記流程、記分類、記誰有保什麼

* **辨識優於記憶**（spec 最強調）
  * 底部／頂部導覽：圖示＋文字並列（總覽／理賠／AI 顧問／保障）
  * 理賠：色彩標籤＋左側色條辨識狀態，單一清單＋標籤色，不用區塊標題分類
  * 保障缺口：成員頭像標示誰有投保
  * AI 顧問：建議功能 Chips、雙入口（對話／情境模擬）

* **系統狀態可見**
  * 理賠：進度環＋百分比；Tab 件數 badge（待處理項）
  * 保障缺口：進度條＋達成率
  * 待辦：總覽日曆紅點、緊急程度色點；成員詳情待辦列表
  * 載入、成功、待補件等明確狀態文案
  * 系統待辦隨保單／缺口狀態即時浮現，與理賠頁讀同一份家庭保單脈絡

* **符合使用者心智**
  * 台灣保險語境（壽險／醫療、同業公會資訊系統／自行登載、理賠階段、續保提醒）
  * 避免抽象代號當主標
  * 續保、待補件、失效等狀態同時出現在**理賠 Tab**與**系統待辦**，語意一致

* **一致性**
  * 全站 m3-card、品牌青綠、HeroUI Modal／Button
  * 保單、缺口、理賠、待辦卡片結構一致
  * 待辦來源可辨識：系統提醒、保障規劃（事件）、手動新增

* **防錯與可恢復**
  * 待辦完成、同業公會資訊系統綁定／取消：二次確認
  * 高風險缺口：「尚未投保」＋明確 CTA
  * 完成續保／補件類待辦後，理賠與保單狀態同步更新，避免各頁資訊不一致

* **減少認知負擔**
  * 漸進揭露：保障總覽點擊才看細項
  * 健康分級：「目前」與「五級總覽」分開
  * 理賠：單一清單＋標籤色，移除區塊標題
  * **不另設獨立「提醒／通知」Tab**；到期、缺口、理賠待辦整合在總覽日曆、成員待辦與理賠頁，減少導覽層級

* **效率與彈性**
  * 熟手：直接點缺口推薦、情境模擬、理賠 Tab 處理待辦項
  * 新手：AI 顧問歡迎引導、建議問題

### 保障體驗連動邏輯（跨頁一致的狀態語言）

* **單一家庭視角**
  * 以全家保單為起點，驅動保障總覽、缺口計算、理賠列表、系統待辦
* **狀態 → 行動（自動浮現）**
  * 保單即將到期 → 續保系統待辦；理賠 Tab 顯示續保提醒
  * 保單待補件／核保中 → 追蹤待辦；理賠 Tab 顯示待補件
  * 保單已失效 → 重新投保待辦；理賠 Tab 顯示失效狀態
  * 保障缺口不足 → 補強系統待辦（如重大疾病尚未覆蓋）
* **行動 → 狀態（完成後同步）**
  * 完成續保待辦 → 保單恢復有效 → 理賠續保提醒消失
  * 完成補件／核保待辦 → 保單恢復有效
  * 手動待辦、家庭事件待辦維持獨立，於總覽日曆與成員詳情操作
* **資訊架構取捨**
  * 移除獨立「提醒」頁；**待辦**分散於總覽（日曆總覽）與保障（成員待辦＋規劃事件）
  * **理賠進度與保單作業**集中於理賠 Tab，避免與待辦重複建第三層通知中心

### Material 3 設計規範：Mobile-first 的導覽與版面骨架，品牌視覺疊加

* **斷點導覽**
  * ≤599px：底部 Navigation Bar（總覽／理賠／AI 顧問／保障）
  * ≥600px：頂部 Pill 導覽
  * 個人資料：右上角頭像進入，不佔主 Tab

* **內容寬度**
  * 一般頁：720px（45rem）置中
  * 總覽：1120px（70rem）置中

### 總結

* **M3 決定：** 手機怎麼逛、資訊放哪、斷點怎麼切
* **Nielsen 決定：** 畫面顯什麼、怎麼標狀態、怎麼少讓使用者記東西
* **連動邏輯決定：** 保單與缺口變了，總覽、理賠、待辦怎麼一起說同一件事

## 4. Interface & Aesthetics（視覺與氛圍定義）

* **Theme:** 淺色系（Light Mode Base）

* **Vibe:**
  * 溫暖、安心、家庭感（溫暖淺色系，避免冷冰冰的保險／金融模板感）
  * 守護、信賴（青綠主色＋「神山」品牌、問候橫幅、成員頭像）
  * 清楚但不嚴肅（圓角、漸層、chip 標籤，理賠用色標而非冷冰冰表格）
  * Mobile-first 生活化（像家庭 App，不像企業後台）

* **色彩策略：** 青綠主導＋暖沙底；珊瑚只點綴；單一淺色主題
  * 主色：青綠 `#2d7a70`（信任、按鈕、導覽、健康度）
  * 輔色：淺青綠 `#d4efec`、輔助綠 `#3d9b8f`
  * 品牌字：深綠 `#0a3d2e`／`#1f5f57`
  * 底色：暖沙 `#faf9f7`（頁面）、白 `#ffffff`（卡片）、邊框 `#e8e4dd`
  * 文字：主 `#2c3338`、次 `#6b7280`
  * 警示：珊瑚 `#d97055`／`#e8917a`（尚未投保、待處理）

## 5. Information Architecture（與現行原型一致）

```
護家神山
├── 1. 總覽
│   ├── 問候橫幅 + 家庭保險健康分級入口
│   ├── 家庭保障總覽（健康度、壽險／醫療分域、缺口、意外理賠）
│   ├── 待辦事件日曆（週／月／年、紅點、詳情 Modal）
│   └── 推薦教育內容
│
├── 2. 理賠
│   ├── 待處理｜進行中｜已完成 分段
│   ├── 進度環 + 狀態標籤（含續保提醒、待補件等）
│   └── 點擊開啟保單詳情 Modal
│
├── 3. AI 顧問
│   ├── 歡迎引導（AdvisorWelcome）
│   ├── 底部 Dock：對話｜情境模擬
│   └── 模擬結果 Modal
│
├── 4. 保障
│   └── 成員儀表板
│       ├── 擁有保單（同業公會資訊系統／自行登載）
│       ├── 待辦提醒與保障規劃事件（同列表、可完成／編輯）
│       └── 已完成事件、文件庫瀏覽
│
└── 個人資料（頭像進入，非 Tab）
    ├── 基本資料、健康分級、文件庫、同業公會資訊系統綁定
    └── 登出示意
```

> **已移除：** 獨立「提醒」Tab 與通知列表頁。保單到期、申請進度等改由**系統待辦**與**理賠狀態**承載。

## 6. Impact & Optimization（影響力與後續優化）

### 可能影響力（Impact）

**用戶／家庭**

* 把分散在各保險公司的保單，收斂成「一家一眼」的保障總覽
* 用缺口進度、壽險／醫療分類，快速看出誰缺什麼、差多少
* 理賠 Tab 集中呈現出險與保單作業進度，降低「不知道進度到哪」的焦慮
* AI 顧問（對話＋情境模擬）降低保障規劃門檻，適合非專業家人討論
* 系統待辦隨保單／缺口自動浮現，家人不必另開提醒頁對照
* 完成待辦後總覽、理賠、保單狀態同步更新，提升跨頁信任感

**產品／商業**

* Mobile-first 適合台灣家庭決策者（子女替父母看、夫妻共同規劃）
* 同業公會資訊系統＋自行登載，貼近真實台灣投保情境
* 家庭保險健康分級可作為後續推薦、顧問媒合的錨點
* 溫暖青綠品牌調性，差異化於冷色金融 App
* 四 Tab 精簡導覽，降低學習成本

### 已實現（互動原型驗證）

* 四 Tab 導覽：總覽／理賠／AI 顧問／保障
* 保單狀態／保障缺口 → 系統待辦自動浮現（總覽日曆＋成員待辦）
* 續保／補件待辦完成 → 保單與理賠狀態連動更新
* 理賠列表依保單狀態動態產生
* 手動待辦、家庭事件待辦與系統待辦並存

### 後續優化方向（Optimization）

**資料與真實性**

* 串接真實同業公會資訊系統／保險存摺資料，取代示範資料
* 保障額、缺口公式與保險公司定義對齊並可稽核
* 理賠與保單狀態與保險公司／同業公會資訊系統即時同步

**功能深化**

* 缺口推薦 → 可比較方案、預約顧問、申請進度追蹤
* 缺口待辦完成 → 串接投保／加保流程
* AI 顧問：真實 LLM、家庭脈絡記憶
* 家庭協作：角色權限、推播通知（FCM／LINE）
* 若產品需要獨立通知中心，可於 P1 評估是否恢復「提醒」入口

**體驗與無障礙**

* WCAG 字級／對比度檢測
* 高齡模式（更大字、更簡路徑）
* Dark mode 或高對比主題

### 對外一句話

> **以家庭視角串起保障總覽、理賠進度與待辦行動，讓家人在同一套介面語言裡看見缺口、理解狀態、知道下一步；本次為互動原型驗證，後續將對接真實保單資料以完善服務體驗。**
